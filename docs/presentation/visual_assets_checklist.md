# Presentation Visual Assets Checklist

Use these screenshots or visuals in a deck.

## Must-Have Screenshots

1. App home screen before analysis.
2. Sidebar with demo repo selected.
3. Overview tab showing project type and framework evidence.
4. AI Analyzer / AI Fusion status if AI is enabled.
5. Architecture graph.
6. Reading path workbench.
7. Component card for `backend/routes/trips.py`.
8. Quiz question.
9. Export tab with Markdown download.

## Optional Screenshots

- ZIP upload safety message.
- Detected routes table.
- Test intelligence tab.
- Work plans and agent workflows.
- Built with IBM Bob sidebar/content.

## Suggested Slide Visuals

- Pipeline diagram:

```text
Repo Input
  -> Safe Scan
  -> Deterministic Evidence
  -> RepoQuest Artifacts
  -> Optional AI Fusion
  -> Onboarding Guide
```

- Trust model:

```text
Facts: deterministic and authoritative
Interpretation: AI can improve with evidence
Unsafe output: rejected or downgraded
```

## Screenshot Tips

- Use the bundled demo repo for clean outputs.
- Zoom browser to 90 percent if the screen is crowded.
- Open one component card and one quiz question before taking screenshots.
- If showing AI Fusion, use mock or local AI so the sidebar shows AI ready.
- Do not show API keys, `.env`, terminal secrets, or provider credentials.
